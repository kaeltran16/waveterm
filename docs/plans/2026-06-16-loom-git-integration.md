# loom Git Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Repurpose the `Cmd:Shift:g` diff-split so it launches the `loom` TUI git client in the active session's repo, with the binary path resolved from a new `app:loombin` config key (default `loom` on PATH).

**Architecture:** One new global config key (`app:loombin`) added the same way as `app:tabbar`. A small pure helper (`loomBinOrDefault`) resolves the configured value with a `"loom"` fallback that also handles a blank string. `keymodel.ts`'s existing `handleDiffSplit` is rebuilt to launch loom interactively (`controller: "cmd"` + `cmd:interactive`) instead of running `git --no-pager diff`, reusing the already-imported `createBlockSplitHorizontally` and `findActiveSessionTermBlock`.

**Tech Stack:** Go (`pkg/wconfig` settings struct), `task generate` codegen, TypeScript/React frontend, Jotai (`getSettingsKeyAtom` global settings read), vitest for the one pure helper.

**Source spec:** `docs/specs/2026-06-16-loom-git-integration-design.md` (Approach A, keybinding-only).

**Conventions for this plan:**
- **No `git commit` steps.** Per the repo owner's strict no-auto-commit rule, each task ends with a **verification checkpoint** (tests + typecheck). The whole change is batched into a single commit made only with explicit approval — matching the Phase 1/2/3 sidebar plans.
- **No `go build`/`go run`.** Per repo rules, Go compilation is confirmed via VSCode problems; the checkpoint for Go edits is that `task generate` succeeds and the generated type appears.
- TDD applies to the one genuinely pure unit (`loomBinOrDefault`). The keybinding + config wiring has no unit-test seam in this codebase (consistent with the prior sidebar phases, which verified keymap/`wsh` wiring live); it is covered by the live checks in Task 4.

**Verified API facts this plan relies on (source-inspected 2026-06-16):**
- `pkg/wconfig/settingsconfig.go:71` — `AppTabBar string json:"app:tabbar,omitempty"` is the field template; `SettingsType` starts at `:59`.
- `pkg/wconfig/metaconsts.go:20` — `ConfigKey_AppTabBar = "app:tabbar"` is the const template.
- `pkg/wconfig/defaultconfig/settings.json:7` — `"app:tabbar": "left"` is where a baked default would go (this plan does **not** bake one).
- Global app settings are read with `globalStore.get(getSettingsKeyAtom("<key>")) ?? <fallback>` (exact pattern at `workspace-layout-model.ts:159` for `app:tabbar`). `app:loombin` has no per-block override, so `getSettingsKeyAtom` (not `getOverrideConfigAtom`) is correct.
- `frontend/app/store/keymodel.ts` already imports `getSettingsKeyAtom` (`:16`), `globalStore` (`:17`), and `createBlockSplitHorizontally` (`:9`) from `@/app/store/global`, and `findActiveSessionTermBlock` (`:24`) from the sidebar model. `BlockDef` is an ambient global type already used by `handleDiffSplit`.
- The interactive-TUI block recipe is `{ view: "term", controller: "cmd", cmd: "<bin>", "cmd:interactive": true, "cmd:cwd": <cwd> }` — same shape the sidebar uses to launch agents (`buildDuplicateBlockMeta` test, `sessionviewmodel.test.ts:376`).
- The current diff-split handler is `handleDiffSplit` at `keymodel.ts:405-419`; its binding is `globalKeyMap.set("Cmd:Shift:g", () => { handleDiffSplit(); return true; })` at `keymodel.ts:562-565`.

**Deviations from the add-config skill (intentional, for fork hygiene):**
- **No `MetaTSType` entry** (skill Step 1.5): the loom binary is a global tool path, not a per-block override.
- **No `docs/docs/config.mdx` row** (skill Step 3): `app:loombin` is a personal-fork setting pointing at a personal binary; documenting it in upstream-facing docs adds rebase surface for no user benefit. Flagged here instead.
- **No baked default in `settings.json`** (skill Step 2): the `loomBinOrDefault` code fallback covers the default, so the key works with zero config and no `settings.json` edit.

- [ ] **Step 1: Write the failing test**

```ts
import { loomBinOrDefault } from "./sessionviewmodel";

describe("loomBinOrDefault", () => {
    it("returns the configured binary when set", () => {
        expect(loomBinOrDefault("loom")).toBe("loom");
        expect(loomBinOrDefault("/usr/local/bin/loom")).toBe("/usr/local/bin/loom");
        expect(loomBinOrDefault("C:\\tools\\loom.exe")).toBe("C:\\tools\\loom.exe");
    });
    it("trims surrounding whitespace", () => {
        expect(loomBinOrDefault("  loom.exe  ")).toBe("loom.exe");
    });
    it("falls back to 'loom' when unset or blank", () => {
        expect(loomBinOrDefault(undefined)).toBe("loom");
        expect(loomBinOrDefault("")).toBe("loom");
        expect(loomBinOrDefault("   ")).toBe("loom");
    });
});
```

- [ ] **Step 3: Write the minimal implementation**

Append to `frontend/app/tab/sessionsidebar/sessionviewmodel.ts` (matches the file's existing `SCREAMING_SNAKE` const style, e.g. `NO_CWD_LABEL`/`COLOR_WORKING`):

```ts
export const DEFAULT_LOOM_BIN = "loom";

// A blank app:loombin value ("" / whitespace) is not null, so it slips past ?? at the call site;
// trim-and-check here so a blank config still falls back to the PATH lookup.
export function loomBinOrDefault(configVal?: string): string {
    const trimmed = configVal?.trim();
    return trimmed && trimmed.length > 0 ? trimmed : DEFAULT_LOOM_BIN;
}
```

- [ ] **Step 2: Replace `handleDiffSplit` with `handleGitSplit`**

Replace the entire current handler (the comment on `:404` through the closing brace on `:419`):

```ts
// git --no-pager diff (not "git diff": the pager would block on the block's TTY)
async function handleDiffSplit() {
    const termBlock = findActiveSessionTermBlock();
    if (termBlock == null) {
        return;
    }
    const blockDef: BlockDef = {
        meta: {
            view: "term",
            controller: "cmd",
            cmd: "git --no-pager diff",
            "cmd:cwd": termBlock.cwd,
        },
    };
    await createBlockSplitHorizontally(blockDef, termBlock.blockId, "after");
}
```

with:

```ts
async function handleGitSplit() {
    const termBlock = findActiveSessionTermBlock();
    if (termBlock == null) {
        return;
    }
    const loomBin = loomBinOrDefault(globalStore.get(getSettingsKeyAtom("app:loombin")));
    const blockDef: BlockDef = {
        meta: {
            view: "term",
            controller: "cmd",
            cmd: loomBin,
            "cmd:interactive": true,
            "cmd:cwd": termBlock.cwd,
        },
    };
    await createBlockSplitHorizontally(blockDef, termBlock.blockId, "after");
}
```
